# Code of Conduct

We want CrisisBrief to be useful, respectful, and welcoming for hackathon participants and reviewers.

## Expected Behavior

- Be kind and constructive.
- Respect different skill levels and backgrounds.
- Give feedback that helps improve the project.
- Avoid sharing private, confidential, or harmful content in demo evidence.

## Unacceptable Behavior

- Harassment, discrimination, or personal attacks.
- Publishing secrets, customer data, or private incident material.
- Misrepresenting generated analysis as verified fact when citations are missing.

## Enforcement

Project maintainers may remove harmful content, close issues, or block participation when needed to keep the project safe.
